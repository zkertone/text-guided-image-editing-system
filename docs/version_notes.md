# 版本迭代记录

## V1.0

### 版本名称

V1.0 基础推理验证版本

### 本次更新内容

本版本完成了毕业设计项目的第一阶段技术可行性验证，初步实现了基于预训练扩散模型的文字驱动图像编辑流程。系统采用 Hugging Face Diffusers 框架调用 InstructPix2Pix 管线，在云端 GPU 环境中成功完成了模型加载、文本指令输入、图像推理生成以及结果输出的基本闭环。

在本阶段中，项目的重点并不在于构建完整界面或实现复杂功能，而是首先验证课题所设定技术路线的可行性，即：在不从零训练模型的前提下，能否基于现有预训练模型完成“输入图像 + 输入文本指令 → 输出编辑结果图像”的核心流程。实验结果表明，该技术路线是可行的，模型能够根据输入的文本语义对原始图像进行一定程度的内容修改与风格调整。

### 解决的问题或达到的阶段

该版本的完成，标志着项目已经从“课题方案与技术设想”正式推进到“可实际运行的原型验证”阶段，为后续用户自定义输入、界面封装、实验留档以及系统工程化整理奠定了基础。

### 下一步计划

在基础推理验证成功的前提下，进一步支持用户自定义图像输入与编辑指令测试，使系统从固定样例验证过渡到面向实际输入场景的功能验证。

## V1.1

### 版本名称

V1.1 用户自定义输入验证版本

### 本次更新内容

本版本在基础示例验证的基础上，进一步实现了用户自定义图像输入与文本指令编辑功能。系统已不再局限于固定测试样例，而是能够接收用户上传的真实图像，并根据输入的英文编辑指令生成对应的图像编辑结果。通过该版本的实现，项目完成了从“官方示例验证”到“用户输入验证”的过渡，说明文字驱动图像编辑流程已经具备基本实用性。

在本阶段实验中，系统完成了人物头发颜色修改、背景变化等初步测试，验证了预训练扩散模型在图像属性调整和场景语义变换方面的可行性。同时也观察到，当编辑目标较复杂时，模型可能对非目标区域产生附带影响，这为后续引入局部编辑与区域约束提供了依据。

### 解决的问题或达到的阶段

该版本的完成，标志着项目已经初步具备“输入图像—输入文本—输出结果”的基础交互能力，为后续工程化整理与界面封装奠定了基础。

### 下一步计划

将当前 notebook 中已验证成功的代码进行函数化封装，并逐步整理为更正式的工程结构，为 Gradio 界面接入和项目模块化做准备。

## V1.2

### 版本名称

V1.2 实验信息可视化与结果留档版本

### 本次更新内容

本版本在 V1.1 的基础上，进一步完善了系统的工程化表达能力与实验记录能力。除了保留原有的图像编辑功能外，系统新增了输入图像与输出图像的自动保存机制，并在 Gradio 页面中增加了实验信息展示区域，用于显示本次实验的输入图保存路径、输出图保存路径以及关键推理参数摘要。

通过这一改进，系统不再只是简单返回编辑后的图像结果，而是具备了基础的实验留档与结果追踪能力。这使得每次图像编辑操作都能够形成较为完整的记录，便于后续进行实验截图整理、参数对比分析以及论文撰写中的过程复现。

### 解决的问题或达到的阶段

该版本的实现，使项目从“可运行 Demo”进一步过渡到“具备基本系统特征的原型”，增强了项目在答辩展示中的完整性，也为后续实验管理和功能扩展提供了更清晰的基础。

### 下一步计划

继续增强系统的实验记录能力，使参数、提示词与结果图能够以结构化方式自动保存，形成可直接用于论文分析和结果整理的实验日志。
